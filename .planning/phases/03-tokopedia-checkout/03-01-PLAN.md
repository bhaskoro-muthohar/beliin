**runTokopediaCheckout(page, sessionId, options)** — async function with try/catch/finally (page.close() in finally). Session state progression:

1. **navigating** — `page.goto(options.url, { waitUntil: 'domcontentloaded' })`. Login wall detection: if URL contains `/login` or `accounts.tokopedia.com`, call `waitForInput(sessionId, 'login_required', 'Please log in to Tokopedia in the browser window, then submit any value to continue')` and wait for URL to change away from login.

2. **selecting_variant** — Look for variant buttons with selector `button[data-testid="pdpVariantOption"], .css-1k4y9ld button, [data-testid="lblPDPVariantName"]`. If variants exist and `options.variant` provided, click matching button. If variants exist but no option provided, collect variant names and call `waitForInput(sessionId, 'variant_selection', ...)`.

3. **adding_to_cart** — Click Buy Now: `button:has-text("Beli Langsung"), button:has-text("+ Keranjang")`. Wait for checkout URL: `page.waitForURL('**/checkout**', { timeout: 15000 })`.

4. **checkout** — `page.waitForLoadState('networkidle')`. Address and shipping are pre-filled (TOKO-02 — no interaction needed).

5. **selecting_payment** — Click "Lihat Semua" to open payment modal: `page.locator('text=Lihat Semua').first().click()`. Then enter the payment-gateway-list iframe (per D-01): `const paymentFrame = page.frameLocator("iframe[title='payment-gateway-list']")`. Inside that iframe, click "Kartu Kredit": `paymentFrame.locator('text=Kartu Kredit').first().click()`. Then click "Pakai Kartu Lain": `paymentFrame.locator('button:has-text("Pakai Kartu Lain")').click()`.

   **Minimum transaction check (per D-04):** Before clicking "Pakai Kartu Lain", check if the button is disabled or if text "Tambah kartu tidak tersedia" is visible inside the payment iframe. If detected, update session to failed with error: `'Card payment unavailable — transaction below minimum (~Rp50,000). Use a higher-value product.'` and return early.

6. **filling_card** — Navigate into the double-nested iframe (per D-01):
   ```typescript
   const cardFrame = paymentFrame.frameLocator('#iframe-creditcard');
   await cardFrame.locator('div#cc-card-no input[data-n-input]').fill(options.card.number);
   await cardFrame.locator('div#cc-exp-date input[data-n-input]').fill(options.card.expiry);
   await cardFrame.locator("button:has-text('Konfirmasi')").click();
   ```
   (TOKO-03 + TOKO-04)

7. **placing_order** — After card tokenization, handle installment modal: wait for "Bayar Penuh" text in the payment iframe (short timeout 5s, proceed if not found), click it if visible. Then close modal / click "Bayar Sekarang" on main page: `page.locator('[data-testid="btnSafChoosePayment"], button:has-text("Bayar Sekarang")').first().click()`.

8. **cvv_entry** (per D-02) — Wait for CVV page: `page.waitForURL('**/payment/validate/**', { timeout: 15000 })`. Fill CVV automatically: `page.locator('input[type="password"], input[type="tel"], input[type="text"]').first().fill(options.card.cvv)`. Click continue: `page.locator('button:has-text("Lanjutkan"), button:has-text("Pay")').first().click()`.

9. **awaiting_payment** (TOKO-05 + TOKO-06) — Check if 3DS triggered by checking URL for `3dsecure` or `acs` patterns. If 3DS: `const otp = await waitForInput(sessionId, 'otp', 'Enter the 3DS OTP sent to your phone')`, fill OTP input, click submit, wait for return to tokopedia.com. If no 3DS: proceed directly.

10. **verifying** — Detect success: `page.locator('text=pembayaran berhasil').first().isVisible({ timeout: 15000 }).catch(() => false)`. If success: update session state to `'success'` with `{ completed_at: Date.now() }`. If not: update to `'failed'` with `{ error: 'Payment result not detected' }`. (TOKO-07)

**Error handling:** catch block updates session to `'failed'` with error message, logs via `console.error('[beliin] Tokopedia checkout failed:', ...)`. Finally block calls `await page.close()`.

All logging via `console.error()` only. Never use `console.log`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCP client -> buy_on_tokopedia tool | Untrusted input (URL, card details) enters via tool parameters |
| Playwright -> tokopedia.com | Browser interacts with external merchant site |
| Playwright -> pay.tokopedia.id iframe | Cross-origin iframe with PCI DSS card form |
| Playwright -> pay.tokopedia.com CVV page | Separate page for CVV entry |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | I (Information Disclosure) | Card details in memory | accept | Card details held in function scope only, not persisted. Virtual cards are single-use. Same risk profile as Phase 2. |
| T-03-02 | T (Tampering) | URL parameter | mitigate | Zod validates `url` as `z.string().url()` — rejects non-URL input at tool entry |
| T-03-03 | S (Spoofing) | Login wall detection | accept | Login handled by human via browser window. No credentials stored or transmitted by beliin. |
| T-03-04 | I (Information Disclosure) | console.error logging | mitigate | Card PAN never logged. Only session IDs and error messages in stderr. |
| T-03-05 | D (Denial of Service) | Timeout on iframe load | mitigate | All waits have explicit timeouts (5-30s). Failure updates session to 'failed' state, page closed in finally block. |
</threat_model>
