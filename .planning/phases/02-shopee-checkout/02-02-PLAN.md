Create `src/merchants/shopee.ts` with the complete Shopee checkout flow.

The module exports one function: `runShopeeCheckout(page, sessionId, options)`. It drives the full flow from product page to payment result, updating session state at each step. It uses a `waitForInput` helper to pause for human input (3DS OTP, variant selection).

```typescript
import { Page } from 'playwright';
import { sessions } from '../lib/session.js';

export interface ShopeeCheckoutOptions {
  url: string;
  card: { number: string; expiry: string; cvv: string; name: string };
  variant?: string;
}

function waitForInput(sessionId: string, inputType: string, prompt: string): Promise<string> {
  return new Promise<string>((resolve) => {
    sessions.update(sessionId, {
      state: 'need_input',
      data: { input_type: inputType, prompt, resolve },
    });
    console.error(`[beliin] Session ${sessionId} paused: ${inputType}`);
  });
}
```

The `runShopeeCheckout` function implements these steps (each step updates session.state):

**Step 1 — Navigate product page** (state: `navigating`):
- `await page.goto(options.url, { waitUntil: 'domcontentloaded' })`
- Check for login wall: if URL contains `/buyer/login`, pause with `waitForInput(sessionId, 'login_required', 'Please log in to Shopee in the browser window, then submit any value to continue')`

**Step 2 — Handle variants** (state: `selecting_variant`):
- Check if variant selectors exist on the product page: `page.locator('.product-variation').or(page.locator('[data-testid="variation"]')).or(page.locator('.flex-column .items-center button'))`
- If variants exist AND `options.variant` is provided: click the button matching the variant text
- If variants exist AND `options.variant` is NOT provided: extract variant names from buttons, pause with `waitForInput(sessionId, 'variant_selection', 'Select a variant: ' + variantNames.join(', '))`. After resume, click the matching variant button. Per D-06.
- If no variants: skip

**Step 3 — Click Buy Now** (state: `adding_to_cart`):
- `await page.locator('button:has-text("Beli Sekarang")').or(page.locator('button:has-text("Buy Now")')).click()`
- Wait for navigation to checkout: `await page.waitForURL('**/checkout**', { timeout: 15000 })`

**Step 4 — Checkout page** (state: `checkout`):
- Address and shipping are pre-filled and auto-selected. Per D-03, do NOT pause.
- Wait for the page to settle: `await page.waitForLoadState('networkidle')`

**Step 5 — Select payment method** (state: `selecting_payment`):
- Click "Credit / Debit Card" payment option: `await page.locator('button:has-text("Kartu Kredit/Debit")').or(page.locator('button:has-text("Credit / Debit Card")')).click()`
- Click "Pay With New Card" or similar: `await page.locator('button:has-text("Kartu Baru")').or(page.locator('button:has-text("New Card")').or(page.locator('button:has-text("Pay With New Card")'))).click()`
- Wait for redirect to card form: `await page.waitForURL('**/payment-v2/add-card**', { timeout: 15000 })`

**Step 6 — Fill card form on pay.shopee.co.id** (state: `filling_card`):
- `await page.waitForLoadState('networkidle')`
- Card Number: `await page.fill("input[type='tel'][maxlength='19']", options.card.number)`
- Expiry: `await page.fill("input[type='tel'][maxlength='5']", options.card.expiry)`
- CVV: `await page.fill("input[type='password'][maxlength='3']", options.card.cvv)`
- Name on Card: `await page.locator("input[type='text'].inputWithStatusInput--nCzAd").first().fill(options.card.name)`
- Per D-07: do NOT touch billing address or postal code fields (pre-filled)
- After each fill, dispatch input event as fallback: `await page.dispatchEvent(selector, 'input')`
- Wait for Submit button to become enabled:
  ```typescript
  await page.waitForFunction(() => {
    const btn = document.querySelector('button.submitBtn--yiKdL');
    return btn && !btn.classList.contains('submitBtnDisabled--djwns');
  }, { timeout: 10000 });
  ```
- Click Submit: `await page.locator('button.submitBtn--yiKdL').click()`
- Wait for redirect back to checkout: `await page.waitForURL('**/checkout**', { timeout: 15000 })`

**Step 7 — Place Order** (state: `placing_order`):
- `await page.waitForLoadState('networkidle')`
- Click Place Order: `await page.locator('button:has-text("Buat Pesanan")').or(page.locator('button:has-text("Place Order")')).click()`

**Step 8 — CVV re-entry** (state: `cvv_entry`):
- Wait for either CVV page or 3DS redirect or success:
  ```typescript
  await page.waitForURL(
    (url) => url.pathname.includes('/payment/') || url.hostname.includes('3dsecure') || url.pathname.includes('/purchase/'),
    { timeout: 30000 }
  );
  ```
- If URL contains `/payment/` (CVV re-entry page):
  - Fill CVV: `await page.fill("input[type='password']", options.card.cvv)`
  - Click Pay: `await page.locator('button:has-text("Pay")').or(page.locator('button:has-text("Bayar")')).click()`
  - Then wait again for 3DS or success

**Step 9 — 3DS or success** (state: `awaiting_payment`):
- Check current URL:
- If URL contains `3dsecure`:
  - Pause for OTP: `const otp = await waitForInput(sessionId, 'otp', 'Enter the 3DS OTP sent to your phone')`
  - Fill OTP: `await page.fill('input[type="text"]', otp)` (or `input[type="password"]`)
  - Submit: `await page.locator('button:has-text("OK")').or(page.locator('button[type="submit"]')).click()`
  - Wait for redirect back: `await page.waitForURL('**shopee.co.id/**', { timeout: 30000 })`
- Per SHOP-06: handle both 3DS-triggered and 3DS-skipped paths

**Step 10 — Payment result** (state: `verifying`):
- Check for success indicators: `page.locator('text=Payment Successful').or(page.locator('text=Pembayaran Berhasil'))`
- If success visible: update session to `state: 'success'` with `data: { completed_at: Date.now() }`
- If not visible after 10s: update session to `state: 'failed'` with `data: { error: 'Payment result not detected' }`

**Error handling:**
- Wrap entire flow in try/catch
- On error: `sessions.update(sessionId, { state: 'failed', data: { error: (err as Error).message } })`
- Log error: `console.error('[beliin] Shopee checkout failed:', (err as Error).message)`
- In finally block: `await page.close()`

CRITICAL constraints:
- All logging via `console.error()` only — never `console.log()`
- Use `page.fill()` for card fields (not `page.type()`) — per RESEARCH anti-patterns
- Use `page.waitForURL()` for navigation detection (not `page.waitForNavigation()`) — deprecated
- Use `page.locator()` API (not `page.$()`) — per RESEARCH state of the art
- Never add manual `sleep()` calls — Playwright auto-waits

Create `src/tools/buy-on-shopee.ts` implementing the buy_on_shopee tool per D-01 and D-02:

```typescript
import { McpServer } from '@modelcontextprotocol/sdk/server/mcp.js';
import { z } from 'zod';
import { sessions } from '../lib/session.js';
import { browser } from '../lib/browser.js';
import { toolResult } from '../lib/types.js';
import { runShopeeCheckout } from '../merchants/shopee.js';

export function registerBuyOnShopeeTool(server: McpServer): void {
  server.tool(
    'buy_on_shopee',
    'Purchase a product on Shopee using a virtual card. Starts checkout asynchronously — use get_session_status to poll progress. Returns session_id immediately.',
    {
      url: z.string().url().describe('Shopee product URL'),
      card_number: z.string().describe('Card PAN (16 digits)'),
      card_expiry: z.string().describe('Card expiry MM/YY'),
      card_cvv: z.string().describe('Card CVV (3 digits)'),
      card_name: z.string().describe('Name on card'),
      variant: z.string().optional().describe('Variant to select (e.g. "Black, Size L"). Omit to be prompted if variants exist.'),
    },
    async ({ url, card_number, card_expiry, card_cvv, card_name, variant }) => {
      try {
        const session = sessions.create('navigating');
        const page = await browser.newPage();

        // Fire-and-forget: start checkout, don't await (D-02)
        runShopeeCheckout(page, session.id, {
          url,
          card: { number: card_number, expiry: card_expiry, cvv: card_cvv, name: card_name },
          variant,
        }).catch((err) => {
          sessions.update(session.id, {
            state: 'failed',
            data: { error: (err as Error).message },
          });
          console.error('[beliin] Shopee checkout failed:', (err as Error).message);
        });

        return toolResult({
          summary: `Checkout started. Session: ${session.id}. Use get_session_status to track progress.`,
          status: 'success',
          data: { session_id: session.id, state: 'navigating' },
        });
      } catch (error) {
        return toolResult({
          summary: `Failed to start checkout: ${(error as Error).message}`,
          status: 'error',
          data: null,
        }, true);
      }
    }
  );
}
```

Key implementation details:
- Per D-01: single tool initiates the full checkout flow
- Per D-02: fire-and-forget — `runShopeeCheckout` is called WITHOUT await, returns session_id immediately
- The `.catch()` handler keeps the Promise chain alive (prevents GC of resolve callbacks — Pitfall 7)
- The `.catch()` handler updates session to 'failed' state on unhandled errors
- Per INFRA-06: outer try/catch returns isError: true if session/page creation fails
- Per INFRA-02: all 6 inputs validated with Zod (url, card_number, card_expiry, card_cvv, card_name, variant optional)

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Information Disclosure | Card details in logs | mitigate | Never log card number, CVV, or expiry to stderr. Only log session state transitions and error messages. |
| T-02-06 | Information Disclosure | Card details in session.data | mitigate | Card details are passed as function args to runShopeeCheckout, NOT stored in session.data. Session only stores state, input_type, prompt, and resolve callback. |
| T-02-07 | Spoofing | Product URL input | mitigate | Zod validates URL format. Playwright navigates in sandboxed browser context. |
